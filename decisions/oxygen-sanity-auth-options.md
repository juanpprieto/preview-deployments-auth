# Auth Strategy Options: Oxygen vs Custom Middleware

## Context

Shopify Oxygen provides auth bypass tokens for private environment deployments. These tokens are deployment-scoped (die on redeploy) and come in two kinds:

- **TESTING_AUTOMATION**: Generated by `npx shopify hydrogen deploy --auth-bypass-token`. Max 12 hours TTL. Available in `h2_deploy_log.json`.
- **USER_SHARED**: Generated by `shopify[bot]` PR comments and the Admin dashboard "Share" button. No expiry. No public API to generate programmatically.

### Why Oxygen Auth Is Insufficient

1. **shopify[bot] only comments on PR preview deploys** -- not on staging or production branch deploys. Post-merge staging has no automated USER_SHARED token source.
2. **12h TESTING_AUTOMATION TTL** is too short for persistent Studio Presentation tool sessions.
3. **Multiple PRs targeting the same branch** cause last-writer-wins on a shared bypass document.
4. **Tokens are deployment-scoped** -- the stable environment URL rejects tokens from other deployments.

These constraints make Oxygen auth unworkable for a CI-driven, multi-environment Sanity Presentation tool integration.

## Option 1: Self-Issued HMAC JWT

### Overview

Replace Oxygen auth with a Hydrogen middleware that validates JWTs signed with HMAC-SHA256. CI generates tokens with configurable TTL on every deploy. Oxygen environments are set to Public.

### Architecture

```
                   ┌─ Oxygen (Public) ─────────────────────┐
                   │                                       │
Request ──────────►│  Hydrogen Middleware                   │
?_preview=JWT      │  ├── Validate JWT (HMAC-SHA256)       │
                   │  ├── Valid? → Continue to app          │
                   │  └── Invalid/missing? → 401           │
                   │                                       │
                   │  React Router 7 App                   │
                   │  └── Normal rendering                 │
                   └───────────────────────────────────────┘
```

### Implementation Details

**Middleware** (`app/middleware/staging-auth.ts`):

```typescript
import {createHmac} from 'node:crypto';

export function validatePreviewToken(request: Request, env: Env): boolean {
  // Skip auth for production environment
  if (!env.STAGING_AUTH_SECRET) return true;

  const url = new URL(request.url);
  const token = url.searchParams.get('_preview')
    || request.headers.get('x-preview-token');

  if (!token) return false;

  // Decode and verify HMAC JWT
  const [header, payload, signature] = token.split('.');
  const expected = createHmac('sha256', env.STAGING_AUTH_SECRET)
    .update(`${header}.${payload}`)
    .digest('base64url');

  if (signature !== expected) return false;

  // Check expiry
  const data = JSON.parse(atob(payload));
  if (data.exp && data.exp < Math.floor(Date.now() / 1000)) return false;

  return true;
}
```

**Token generation** (CI, in deploy workflow):

```bash
# Generate JWT with 30-day TTL
PAYLOAD=$(echo -n "{\"iat\":$(date +%s),\"exp\":$(($(date +%s)+2592000)),\"env\":\"staging\"}" | base64 -w0 | tr '+/' '-_' | tr -d '=')
HEADER=$(echo -n '{"alg":"HS256","typ":"JWT"}' | base64 -w0 | tr '+/' '-_' | tr -d '=')
SIGNATURE=$(echo -n "${HEADER}.${PAYLOAD}" | openssl dgst -sha256 -hmac "$STAGING_AUTH_SECRET" -binary | base64 -w0 | tr '+/' '-_' | tr -d '=')
TOKEN="${HEADER}.${PAYLOAD}.${SIGNATURE}"
```

**Env vars:**

| Variable | Scope | Value |
|---|---|---|
| `STAGING_AUTH_SECRET` | Staging, Preview, Dev | Random 256-bit key |
| (not set) | Production | Middleware auto-skips |

### CI Flow

```
Push to any branch
  → Oxygen deploy (env = Public)
  → Generate 30-day JWT signed with STAGING_AUTH_SECRET
  → Store in Sanity: {_id: "auth.pr-{N}", token, deploymentUrl, branch, prNumber}

Merge to staging
  → Oxygen deploy (env = Public)
  → Generate 30-day JWT
  → Store in Sanity: {_id: "auth.staging", token, stableUrl, deploymentUrl}
  → No dependency on shopify[bot]
```

### Studio User Flow

```
1. Editor opens Presentation tool in Studio
2. Studio's async `previewUrl.initial({client})`:
   → Queries Sanity: *[_type == "auth.stagingPreview"][0]{deploymentUrl, token}
   → Returns: ${deploymentUrl}?_preview=${token}
3. Iframe loads → Hydrogen middleware validates JWT → site renders
4. Visual editing works normally
5. Branch picker (custom header) lists all auth.* docs → editor switches
```

### External Stakeholder Flow

```
1. Stakeholder receives link: staging-xxx.myshopify.dev?_preview=eyJhb...
2. Browser navigates → middleware validates JWT → site renders
3. Token works for 30 days across all deploys to the same environment
4. No login, no form, instant access with the link
```

## Option 2: Auto-TOTP + Session JWT

### Overview

Replace Oxygen auth with a Hydrogen middleware that validates TOTP codes and issues session JWTs. A shared TOTP seed is stored in env vars and in Sanity. Studio auto-generates TOTP codes transparently. Stakeholders enter codes manually. Oxygen environments are set to Public.

### Architecture

```
                   ┌─ Oxygen (Public) ──────────────────────┐
                   │                                        │
Request ──────────►│  Hydrogen Middleware                    │
                   │  ├── Has valid session JWT? → Continue  │
                   │  ├── Has valid TOTP code?               │
                   │  │   └── Yes → Issue session JWT        │
                   │  │       → Redirect with ?_session=JWT  │
                   │  └── No auth → Return TOTP entry form   │
                   │                                        │
                   │  React Router 7 App                    │
                   │  └── Normal rendering                  │
                   └────────────────────────────────────────┘
```

### Implementation Details

**Middleware** (`app/middleware/staging-auth.ts`):

```typescript
import {authenticator} from 'otplib';

export function validateStagingAuth(request: Request, env: Env): Response | null {
  // Skip auth for production
  if (!env.STAGING_TOTP_SEED) return null;

  const url = new URL(request.url);

  // Check for existing session JWT
  const sessionToken = url.searchParams.get('_session')
    || getCookie(request, 'staging_session');
  if (sessionToken && validateSessionJwt(sessionToken, env.STAGING_SESSION_SECRET)) {
    return null; // Authenticated, continue
  }

  // Check for TOTP code (from Studio auto-TOTP or form submission)
  const totpCode = url.searchParams.get('_totp');
  if (totpCode && authenticator.verify({token: totpCode, secret: env.STAGING_TOTP_SEED})) {
    // Issue session JWT (30 days)
    const jwt = createSessionJwt(env.STAGING_SESSION_SECRET, 30 * 24 * 60 * 60);
    // Redirect back with session token (for iframe cookie bypass)
    const redirectUrl = new URL(url);
    redirectUrl.searchParams.delete('_totp');
    redirectUrl.searchParams.set('_session', jwt);
    return Response.redirect(redirectUrl.toString(), 302);
  }

  // No valid auth → return TOTP entry form
  return new Response(renderTotpForm(url), {
    status: 401,
    headers: {'Content-Type': 'text/html'},
  });
}
```

**Env vars:**

| Variable | Scope | Value |
|---|---|---|
| `STAGING_TOTP_SEED` | Staging, Preview, Dev | Base32-encoded TOTP seed |
| `STAGING_SESSION_SECRET` | Staging, Preview, Dev | Random 256-bit key for session JWT signing |
| (neither set) | Production | Middleware auto-skips |

### CI Flow

```
Push to any branch
  → Oxygen deploy (env = Public)
  → Store in Sanity: {_id: "auth.pr-{N}", deploymentUrl, branch, prNumber}
  → No token needed — TOTP seed handles auth

Merge to staging
  → Oxygen deploy (env = Public)
  → Store in Sanity: {_id: "auth.staging", stableUrl, deploymentUrl}
  → No token needed — TOTP seed handles auth
```

### Studio User Flow (Zero Friction)

```
1. Editor opens Presentation tool in Studio
2. Studio's async `previewUrl.initial({client})`:
   → Queries Sanity for deployment URL
   → Returns: ${deploymentUrl}
3. Studio's `previewMode.enable` action:
   → previewMode function reads TOTP seed from Sanity doc
   → Generates current TOTP code using otplib
   → Constructs URL: /api/preview-mode/enable?_totp=${code}&sanity-preview-secret=xxx
4. Hydrogen middleware validates TOTP → issues session JWT → redirects
5. Iframe reloads with ?_session=JWT → site renders
6. Fetch interceptor appends _session to all subsequent requests
7. Visual editing works normally
8. Branch picker (custom header) lists auth.* docs → editor switches
```

### External Stakeholder Flow

```
1. Stakeholder opens staging-xxx.myshopify.dev
2. Sees TOTP entry form (styled, branded)
3. Opens authenticator app (Google Auth, Microsoft Auth, Cisco Duo mobile)
4. Enters 6-digit code
5. Middleware validates → issues 30-day session JWT cookie
6. Site renders, session persists for 30 days
7. Return visits: session cookie is valid, no re-auth
```

### TOTP Seed Distribution

```
1. Generate seed: npx -y otplib-cli generate-secret
2. Store in Oxygen env vars: STAGING_TOTP_SEED=JBSWY3DPEHPK3PXP
3. Store in Sanity doc for Studio previewMode to read
4. Share QR code with team via internal channel (Slack, Teams, email)
5. Each team member scans into their authenticator app
6. Codes rotate every 30 seconds — standard RFC 6238
```

## Shared Architectural Elements

Both Option 1 and Option 2 share these design decisions:

### Oxygen Environment Must Be Public

Setting the Oxygen environment to Public removes Oxygen's auth layer entirely. The Hydrogen middleware becomes the sole auth gate. This is safe because:

- The middleware runs before any route handler
- Unauthorized requests get 401 (not a 302 redirect to Shopify OAuth)
- Production environment remains Public with no middleware (env vars not set)
- The middleware is the auth layer, not Oxygen

### Per-PR Bypass Documents in Sanity

```
oxygen-bypass.pr-13  → {deploymentUrl, token?, branch: "feat/staging-diff", prNumber: 13}
oxygen-bypass.pr-14  → {deploymentUrl, token?, branch: "feat/new-feature", prNumber: 14}
oxygen-bypass.staging → {stableUrl, deploymentUrl, token?}
```

Schema changes: add `prNumber` (number), `headBranch` (string), `baseBranch` (string) to the bypass doc type.

### Studio Branch Picker (Custom Header Component)

Both options use the same Studio extensibility:

- `components.unstable_header` renders a dropdown in the Presentation tool header
- Dropdown queries `*[_type == "sanity.oxygenProtectionBypass"]` for all active deployments
- `usePresentationNavigate()` navigates the iframe to the selected deployment
- Auto-selects most recent, with manual override

### CI Sync Workflow Changes

```yaml
# Before: hardcoded single doc
_id: "oxygen-bypass.staging"

# After: per-PR doc with metadata
_id: "oxygen-bypass.pr-${{ github.event.issue.number }}"
branch: "$HEAD_BRANCH"
baseBranch: "$BASE_BRANCH"
prNumber: ${{ github.event.issue.number }}
```

Additional workflow for staging branch deploys:

```yaml
on:
  push:
    branches: [staging]
# Reads h2_deploy_log.json directly (no shopify[bot] dependency)
# Writes to oxygen-bypass.staging with stable URL + JWT/no-token
```

### Cleanup Workflow

```yaml
on:
  pull_request:
    types: [closed]
# Deletes oxygen-bypass.pr-{N} from Sanity when PR is closed/merged
```

## Plan B: 12h Rolling Token via Cron

If neither Option 1 nor Option 2 is viable, a hacky but automated fallback:

- Keep Oxygen auth (Private environments)
- CI cron job re-deploys staging every 11 hours
- Each deploy produces a fresh 12h TESTING_AUTOMATION token
- Sync workflow writes it to Sanity
- Studio always has a valid token (max 1h stale window)
- Downside: 11 unnecessary deploys per day, fragile, wasteful

This is documented for completeness. Not recommended.
